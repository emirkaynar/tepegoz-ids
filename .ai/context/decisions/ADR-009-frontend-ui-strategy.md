# ADR-009: Frontend UI Strategy (Grafana App Plugin + Headless FastAPI)

## Status

Accepted

## Context

The system requires a unified, single-URL experience for the user. We need read-only visualization (which Grafana handles perfectly) as well as interactive configuration menus (tuning rule thresholds, etc.). Previously, an iframe-based wrapper was considered, but it introduced security risks (unauthenticated local data exposure) and poor UX (potential double logins).

Using third-party community plugins (like deprecated form panels) for configuration input is unreliable. We need a native vector to inject our configuration forms directly into the Grafana environment without compromising the CPU budget of the packet sniffer.

## Decision

We will build a **Custom Grafana App Plugin** (React/TypeScript) to serve as the configuration UI natively within Grafana.
The configuration payloads generated by this plugin will be sent to a headless **FastAPI backend**, which remains isolated in a completely separate Docker container.

## Consequences

- **Native UX & Security**: The user stays entirely within Grafana. We leverage Grafana's native authentication, avoiding bypass risks and double-login scenarios.
- **Backend Isolation Maintained**: Process isolation ensures the configuration API (FastAPI) cannot steal CPU cycles from the packet sniffer. Grafana will use its internal proxy functionality to securely route REST calls to the FastAPI container without exposing the API to the public network.
- **Increased Frontend Complexity**: We must write, build, and maintain a React-based plugin using `@grafana/ui` and `@grafana/create-plugin` instead of simple HTML templates.
- **Deployment Mechanics**: The compiled App Plugin will need to be volume-mounted into the Grafana container (`/var/lib/grafana/plugins`) via Docker Compose.

## Implementation Guidance

1. **Plugin Architecture:**
    - Scaffold the UI using `@grafana/create-plugin` (App Plugin type).
    - Utilize `@grafana/ui` components to ensure the forms match the native dashboard aesthetic.
2. **Data Proxying:**
    - Configure the plugin to use Grafana's backend proxy feature.
    - Define a data source or direct API route in the plugin settings so that HTTP POSTs (e.g., saving `rules.yaml`) flow from the browser -> Grafana Backend -> Headless FastAPI container.
3. **Authentication:**
    - Grafana authentication can remain fully enabled.
    - The FastAPI backend does not need user authentication logic, as it will only accept traffic routed internally via the Grafana Docker network proxy.
